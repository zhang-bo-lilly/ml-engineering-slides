## Key definitions (used across both slides)

**Effective quota (EQ):**  
`EQ = allocated GPU-hours / evaluation period hours`  
Always computable. Expressed in the same unit as quota (GPUs), so it can be compared directly against Q (the squad's quota).

**ND** = new demand (GPUs requested by the squad for the next cycle)  
**DF** = discount factor (applied to over-quota allocation in Tier 2)  
**Q** = squad's current quota (GPUs)

---

## Tier system (corrected — EQ vs Q, no division by zero in classification)

| Tier | Condition | Weight formula |
|---|---|---|
| 1 | EQ > Q **AND** avg utilization > 80% | `EQ + ND` |
| 2 | EQ > Q **AND** avg utilization ≤ 80% | `(EQ − Q) × DF + Q + ND` |
| 3 | 0.8 × Q ≤ EQ ≤ Q | `Q + ND × (EQ / Q)` |
| 4 | EQ < 0.8 × Q | `EQ` |

**Division safety for Tier 3:** If Q = 0 and EQ > 0, the squad is in Tier 1 or 2 (no division). If Q = 0 and EQ = 0 (no allocation, no history), the squad is treated as a new squad (see below).  
**New squads (no history):** default to Tier 3 with `weight = min(ND, new project max weight)`.

**Final quota calculation:**  
`New quota = (squad weight / Σ all weights) × available GPU pool`  
`Available GPU pool = Total GPUs − Common Pool`
