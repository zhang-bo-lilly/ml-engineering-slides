# CLAUDE.md

This repo holds AIOS team meeting slide decks. Each presentation is in its own `MM-DD-YYYY/` directory.

## Creating presentations

Use the `/aios-slides` slash command. Work one presentation at a time, one slide at a time.

## Directory conventions

- Source files to commit: `*.py`, `*.md`, static assets in `assets/` (`*.pdf`, PNGs not generated by scripts)
- Generated files (never commit): `*.png` produced by scripts, `*.pptx`, `*.docx`, `.venv/`, `__pycache__/`

## Python environment

A shared venv lives at `~/.claude/venvs/aios-slides/`. Activate it at the start of every session — never create a per-project venv.

```bash
source ~/.claude/venvs/aios-slides/bin/activate
```

If a script needs a package not yet installed, add it to the shared venv there.

## Script path conventions

Never hard-code absolute paths. All scripts must resolve paths relative to the script file itself:

```python
import os
BASE_DIR = os.path.dirname(os.path.abspath(__file__))
```

Use `os.path.join(BASE_DIR, 'filename.png')` for all file reads and writes.

Static input assets (icons, reference images not produced by scripts) go in `assets/` inside the presentation directory. Reference them as `os.path.join(BASE_DIR, 'assets', 'filename.png')`. The `assets/` folder is exempt from the `*.png` gitignore rule.

## gen_slides.py must auto-run diagram generators

`gen_slides.py` is the single entry point — the user should never have to run diagram scripts manually. At the top of the `if __name__ == '__main__'` block, call each diagram generator via subprocess before building slides:

```python
import subprocess, sys

if __name__ == '__main__':
    for script in ['diagram_foo.py', 'diagram_bar.py']:
        subprocess.run([sys.executable, os.path.join(BASE_DIR, script)], check=True)
    # then call slide_01(), slide_02(), ...
```

## Cleaning

Run `./clean.sh` from repo root to delete all generated artifacts. Use `./clean.sh --venv` to also remove virtual environments.
